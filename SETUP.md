The geo-router resolves the visitor's country via `/api/ip-country` (geoip-lite),
then dispatches `lb:regionChanged`. `region.js` re-renders prices, contact info,
and any element marked `data-region-only="NP"` / `data-region-hide="NP"`.

- **Nepal IP вҶ’** Nepal site (NPR pricing, eSewa / Khalti / Nabil / COD, Pathao logistics).
- **Other recognized region (AU/US/GB/EU/CA/NZ/JP/CN) вҶ’** that region's currency + payment + carrier.
- **Anything else (bots, private IPs, lookup failures) вҶ’** Australia fallback (AUD, Stripe / PayPal, AusPost).

## 1. Install

```bash
# Project root
npm install

# Admin dashboard (React)
npm --prefix admin-dashboard install

# Backend API (Express)
npm --prefix backend install
```

## 2. Configure environment

Run the interactive setup вҖ” it writes `backend/.env` and auto-generates strong JWT secrets:

```bash
npm run setup
```

Re-run any time to update keys. Existing values are preserved unless you type a new one.

For CI / Docker (no prompts, just fill safe defaults + JWT secrets):

```bash
npm run setup:silent
```

You can also edit `backend/.env` by hand. A reference of every supported variable is in
`backend/.env.example`.

### Minimum keys to ship

| Variable | Why | Where to get it |
|---|---|---|
| `JWT_SECRET`, `JWT_REFRESH_SECRET` | Admin login | Auto-generated by `npm run setup` |
| `DATABASE_URL` | Persist orders, products | Render Postgres / your DB host |
| `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET` | Card / Apple Pay / Afterpay checkout | https://dashboard.stripe.com/apikeys |
| `PAYPAL_CLIENT_ID`, `PAYPAL_CLIENT_SECRET` | PayPal checkout | https://developer.paypal.com/dashboard/applications |
| `SENDGRID_API_KEY` | Order confirmation emails | https://app.sendgrid.com/settings/api_keys |
| `SITE_URL`, `FRONTEND_URL` | Payment return / cancel URLs | your production domain |

### Nepal (only needed if you take NP traffic)

| Variable | Purpose |

First admin login uses `admin` / `DEFAULT_ADMIN_PASSWORD` (set during `npm run setup`).

## 4. How products flow Admin вҶ’ Storefront

1. Admin signs in at `/admin` вҶ’ JWT issued (`/api/auth/login`).
2. **Products** page uses the `Authorization: Bearer <token>` header.
3. Image upload: `POST /api/upload/image` (multer disk storage, 8 MB cap, JPEG/PNG/WebP/AVIF) вҶ’
   returns `{ url: "/uploads/products/<file>" }`.
4. Create / update product: `POST /api/products` вҶ’ row in Postgres `products` table.
5. Storefront calls `GET /api/products` (or `/api/store` JSON-store fallback when DB is offline).
6. `pricing.js` formats per-region prices; `region.js` re-renders product cards on `lb:regionChanged`.

There is **no third dual-storage path** вҖ” DB-backed and JSON-store flat-file routes both expose
the same shape. The JSON store is used only when `DATABASE_URL` is missing (dev / first boot).

## 5. Region & currency UX

- **`geo-router.js`** вҖ” IP вҶ’ region code (NP, AU, US, GB, EU, CA, NZ, JP, CN). Falls back to AU.
- **`region.js`** вҖ” listens for `lb:regionChanged`, updates flags, contact info, hero copy,
  product card prices, and toggles `data-region-only` / `data-region-hide` elements.
- **`au-currency-converter.js`** вҖ” adds a "View prices in:" dropdown that converts the
  region's base price using live FX (open.er-api.com вҶ’ frankfurter.app fallback).
- **`pricing.js`** вҖ” hardcoded per-region price for every product, with a `DEFAULT` fallback.

To gate any block of HTML to a region, mark it up:

```html
<div data-region-only="NP">вҖҰshown only to Nepal visitorsвҖҰ</div>
<div data-region-only="AU,US,GB,EU,CA,NZ,JP">вҖҰshown to non-Nepal visitorsвҖҰ</div>
<div data-region-hide="NP">вҖҰshown to everyone except NepalвҖҰ</div>
```

The `<body>` tag also gets a `region-{CODE}` class (e.g. `region-NP`) so CSS can react.

## 6. Deployment

### Storefront вҶ’ Vercel
- `vercel.json` rewrites `/api/*` to `api/__proxy.js` which forwards to `BACKEND_URL`
  (set in Vercel project settings to `https://YOUR-SERVICE.onrender.com`).
- Build runs `node scripts/build.js`, which builds the admin dashboard and copies static
  files into `public/`.

### Backend вҶ’ Render
- `render.yaml` defines: web service (Express :3001), worker service (queue), Postgres 16, Redis 7.
- Set every secret in the Render dashboard вҶ’ **Environment**. Render's `sync: false` flag in
  `render.yaml` ensures secrets are never committed to git.

### Health checks
- `GET /api/health` вҖ” JSON status + WS client count.
- `GET /api/health/metrics` вҖ” heap / RSS / uptime, Prometheus-compatible JSON.

## 7. Common issues

| Symptom | Cause / Fix |
|---|---|
| Login returns "email: Required" | Vite dev proxy is hitting the wrong port. From repo root: `npm run backend:dev` (port 3010), then `npm run dev:admin`. |
| `STRIPE_SECRET_KEY not configured` | Run `npm run setup` and pick "configure Stripe". |
| Orders disappear on restart | `DATABASE_URL` missing вҶ’ API uses in-memory store. Set the var. |
| Region detection always returns AU | Bot user-agent or private IP вҖ” that's by design. Override in DevTools by calling `GeoRouter.set('NP')`. |
| Admin image upload 401 | JWT expired вҖ” refresh the page or log out and back in. |
| `/api/upload/image` returns 403 | Logged-in user is not `owner` or `manager` вҖ” promote them in the `users` table. |

## 8. Where the secrets actually go

- Local dev: `backend/.env` (created by `npm run setup`, file permissions `0600`, gitignored).
- Render: Dashboard вҶ’ service вҶ’ Environment.
- Vercel: Dashboard вҶ’ project вҶ’ Settings вҶ’ Environment Variables (`LWB_API_BASE`, `BACKEND_URL`,
  `VITE_API_URL` for the admin SPA build).

Never commit `.env`. `backend/.gitignore` already excludes it.
